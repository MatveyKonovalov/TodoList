# 📋 TodoList

**TodoList** — это современное Android-приложение для управления задачами, построенное на **Jetpack Compose** с использованием **Clean Architecture**. Оно позволяет легко добавлять, отсматривать, редактировать и удалять задачи, а также фильтровать их по датам.

## ✨ Основной функционал

*   **Управление задачами (CRUD):** Полный цикл работы с задачами — создание, просмотр, редактирование и удаление.
*   **Календарь и фильтрация по дате:** Интуитивно понятный недельный календарь для быстрой навигации и просмотра задач на выбранный день.
*   **Приоритеты и категории:** Возможность назначать задачам приоритет (Лёгкий, Средний, Сложный) и категорию для лучшей организации.
*   **Современный UI:** Адаптивный интерфейс, построенный на Jetpack Compose с поддержкой темной темы.

## 🛠 Используемые технологии и библиотеки

Проект использует современный стек технологий, обеспечивающий производительность, масштабируемость и удобство разработки:

*   **Язык:** Kotlin — основной язык разработки, Java — основной язык тестов.
*   **UI Фреймворк:** [Jetpack Compose](https://developer.android.com/jetpack/compose) — для декларативного построения пользовательского интерфейса.
*   **Архитектура:** [Clean Architecture](https://developer.android.com/topic/architecture) — разделение на слои `data`, `domain`, `presentation` для гибкости и тестируемости.
*   **Локальная база данных:** [Room](https://developer.android.com/topic/libraries/architecture/room) — для хранения задач на устройстве.
*   **Внедрение зависимостей:** [Dagger Hilt](https://dagger.dev/hilt/) — для упрощения DI и управления зависимостями.
*   **Асинхронность:** [Coroutines](https://kotlinlang.org/docs/coroutines-overview.html) и [Flow](https://developer.android.com/kotlin/flow) — для асинхронных операций и реактивного программирования.

## 🏗 Архитектура проекта

Проект следует принципам **Clean Architecture**, что обеспечивает четкое разделение ответственности:

*   **`data` слой:** Отвечает за работу с данными. Содержит реализацию Room (сущности `TaskEntity`, DAO, база данных), мапперы для преобразования между `Entity` и `Domain` моделями, а также репозиторий, который является источником данных для верхних слоев.
*   **`domain` слой:** Содержит бизнес-логику приложения. Здесь определены основные модели данных, такие как `Task`.
*   **`presentation` слой:** Отвечает за отображение данных и взаимодействие с пользователем. Содержит `ViewModel` (хранит состояние и обрабатывает события UI) и Compose-экраны (`MainScreen`) и переиспользуемые компоненты (`components`).

## 🗄 Структура базы данных

| Таблица | Поле | Тип | Описание |
| :--- | :--- | :--- | :--- |
| **tasks** | `id` | `Long` | Первичный ключ, автоинкремент |
| | `title` | `String` | Название задачи |
| | `description` | `String` | Описание задачи |
| | `priority` | `String` | Приоритет (Easy, Medium, Hard) |
| | `category` | `String` | Категория (Работа, Личное и т.д.) |
| | `isComplete` | `Boolean` | Статус выполнения задачи |
| | `date` | `String` | Дата, к которой привязана задача |

## 🚀 Планы по развитию

Проект активно развивается. В ближайших планах:

*   **Полное покрытие тестами:** Написание тестов для presentation слоя.
*   **Новые UI-фичи и жесты:**
    *   **Swipe-to-Delete/Edit:** Удаление или редактирование задачи смахиванием (swipe) в списке.
    *   **Pull-to-Refresh:** Обновление списка задач смахиванием вниз.
    *   **Анимации:** Добавление плавных анимаций при добавлении, удалении и завершении задач.
